User hierarchy

Admin vs Customer responsibilities

Both roles inherit from the abstract User class and rely on login() / logout().

Admin

<<User>> adminRole: string

  • Moderate reviews via moderateReview()
  • Inspect every order with viewAllOrders()
  • Audit payments and refunds before Statistic updates
Full access

Customer

customerType: string

  • Interact through register() & view orders
  • Create tickets, merchandise orders, and reviews
  • Trigger payments and receipts for every purchase
